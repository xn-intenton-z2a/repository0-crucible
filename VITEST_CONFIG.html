Configuring Vitest

Configuration File Resolution

  • Priority: vitest.config.ts/js > CLI --config > process.env.VITEST or mode property
  • If no vitest.config present, reads root vite.config.ts under Vite conventions

Supported Config File Extensions

.js, .mjs, .cjs, .ts, .cts, .mts (no .json)

defineConfig API Signature

import { defineConfig } from 'vitest/config'
export default defineConfig({
  test: {
    // options
  }
})

mergeConfig API Signature

import { defineConfig, mergeConfig } from 'vitest/config'
import viteConfig from './vite.config'
export default mergeConfig(
  viteConfig,
  defineConfig({
    test: {
      // overrides
    }
  })
)

Workspace Support

import { defineWorkspace } from 'vitest/config'
export default defineWorkspace([
  'packages/*',
  'tests/**/vitest.config.{e2e,unit}.ts',
  { test: { name: 'node', environment: 'node', setupFiles: ['./setup.node.ts'] } }
])

CLI Options (partial)

  • --config : Specify config file
  • -w, --watch : Enable watch mode (default: !process.env.CI)
  • -u, --update : Update snapshots (default: false)
  • --port : Specify server port
  • --root , --dir : Define project root and scan directory
  • --environment : Set test environment (node|jsdom|happy-dom|edge-runtime)
  • --pool : Set pool type (threads|forks|vmThreads|vmForks)
